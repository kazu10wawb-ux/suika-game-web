# suika-game-web

①制作物の概要、特徴、機能
概要：ターミナル上で動作していたCUIベースのスイカ割りゲームを、PythonとDjangoを用いてWebアプリとして再構築しました。

特徴：文字入力による操作から、ブラウザ上のボタン操作に変更し、直感的に操作できるようにしました。また、プレイ結果をデータとして保存し、直近の結果を表示できるようにしています。

主な機能：
・ブラウザ上のボタンによるプレイヤーの移動
・プレイヤーとスイカの位置関係に応じたヒント表示
・SQLiteを用いたスコア（名前・歩数・日時）の保存
・保存したデータを基にしたランキング表示
②開発の背景
大学の授業課題として、既存のCUIゲームを改良する課題があり、
より分かりやすく操作できる形にしたいと考え、Webアプリ化に取り組みました。

また、ゲームの結果を保存できるようにすることで、
単発で終わらない仕組みを作りたいと考えたことも動機の一つです。
③制作物の構成、アーキテクチャ
開発言語：Python
フレームワーク：Django
データベース：SQLite

Djangoを用いて、画面表示・処理・データ管理を分けて構成しています。

また、セッション機能を利用することで、
ページを更新してもプレイヤーの位置などの状態が保持されるようにしました。
④開発期間・時間
開発期間：約2週間
総作業時間：約5時間
⑤工夫した点・課題点等
工夫した点は、プレイヤーの位置やゲームの進行状況をセッションで管理し、
ページ遷移後も状態が維持されるようにした点です。

また、スコアをデータベースに保存することで、
ランキングとして表示できるようにしました。

課題としては、現在はボタン操作のたびに画面が更新されるため、
操作にやや遅れを感じる点があります。

今後は、画面の更新を減らす工夫を行い、
よりスムーズに操作できるよう改善したいと考えています。


## 実行方法

本アプリケーションはローカル環境で動作します。

1. リポジトリをクローン
git clone https://github.com/kazu10wawb-ux/suika-game-web

2. ディレクトリに移動
cd suika-game-web

3. 必要なパッケージをインストール（必要に応じて）
pip install django

4. サーバーを起動
python manage.py runserver

5. ブラウザで以下にアクセス
http://127.0.0.1:8000/

## 補足

本アプリはDjangoを用いて開発しているため、
ローカル環境で動作する構成となっています。

そのため、Web上で直接操作できる公開URLはありませんが、
GitHub上のソースコードおよび上記手順により動作確認が可能です。
